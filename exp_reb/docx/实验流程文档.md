## 2. 核心概念：模型版本

### 2.1 模型版本定义

基于 Excel 表格中的数据，每个 AI 服务可以有不同的模型版本：

```
服务 s 有多个模型版本可用:

┌─────────────────────────────────────────────────────────────────┐
│  模型版本    │ μ (处理速率) │ 精度(accuracy) │ model_params │ FLOPs │
├─────────────┼─────────────┼────────────────┼──────────────┼──────┤
│  Model-H    │    低 (慢)   │     高         │    大        │  高  │
│  Model-M    │    中等      │     中等       │    中等      │  中  │
│  Model-L    │    高 (快)   │     低         │    小        │  低  │
└─────────────────────────────────────────────────────────────────┘

处理速率 μ: 每秒能处理的请求数
精度: 任务性能指标 (accuracy, mIoU, SSIM等)
```

### 2.2 模型参数映射 (参考 Excel 数据)

以 class_scene 任务为例：

| 模型版本 | 典型 μ (请求/秒) | 典型精度 | 说明 |
|---------|-----------------|---------|------|
| Model-H | 5 | 62% | 高精度，复杂模型，处理慢 |
| Model-M | 10 | 53% | 中等精度，中等复杂度 |
| Model-L | 20 | 45% | 低精度，简单模型，处理快 |

```python
# 模型版本配置示例
MODEL_VERSIONS = {
    "Model-H": {"mu": 5, "accuracy": 0.62, "cpu": 2, "gpu": 2048},
    "Model-M": {"mu": 10, "accuracy": 0.53, "cpu": 1, "gpu": 1024},
    "Model-L": {"mu": 20, "accuracy": 0.45, "cpu": 1, "gpu": 512},
}
```

### 2.3 关键洞察

```
负载 λ 高时 (λ=100):
├── 选择 Model-L (μ=20) → ρ = 100/(c*20) = 合理 → 延迟可接受
└── 选择 Model-H (μ=5) → ρ = 100/(c*5) = 过高 → 排队延迟爆炸

负载 λ 低时 (λ=10):
├── 选择 Model-H (μ=5) → ρ = 10/(c*5) = 低 → 精度高
└── 选择 Model-L (μ=20) → ρ = 10/(c*20) = 更低 → 资源浪费
```

## 3. 算法策略差异

### 3.1 Our 算法 (动态模型选择)

```python
class OurAlgorithm(DeploymentAlgorithm):
    """
    Our算法: 根据负载和约束动态选择模型版本

    决策变量:
    - 部署位置 (节点)
    - 模型版本 (高/中/低)

    目标:
    - 满足延迟约束
    - 最大化精度 (或满意度)
    - 最小化资源使用
    """

    def deploy(self, topology, services, chains) -> DeploymentPlan:
        # 1. 对每条链，分析负载 λ
        # 2. 根据 λ 决策每个服务使用哪个模型版本
        #    - 高负载 → 选低精度高μ的模型 (Model-L)
        #    - 低负载 → 选高精度低μ的模型 (Model-H)
        # 3. 部署到合适节点
        pass
```

**Our 算法决策逻辑**:
```
对于服务 s, 负载 λ:
  if λ > threshold_high:
      选择 Model-L (高μ, 低精度, 满足延迟)
  elif λ < threshold_low:
      选择 Model-H (低μ, 高精度, 资源充裕)
  else:
      选择 Model-M (平衡)
```

### 3.2 Baseline 算法 (固定中等模型)

```python
class BaselineAlgorithm(DeploymentAlgorithm):
    """
    Baseline算法: 固定使用中等模型，不动态选择

    所有服务统一使用 Model-M:
    - μ = 10 (中等)
    - accuracy = 0.53 (中等)
    """

    def deploy(self, topology, services, chains) -> DeploymentPlan:
        # 所有服务固定使用 Model-M
        # 只决策部署位置，不改变模型版本
        pass
```

**Baseline 策略对比**:

| 算法 | 模型选择策略 | 说明 |
|------|-------------|------|
| Our | 动态选择 | 根据负载选择 H/M/L |
| FFD-M | 固定Model-M | 部署用FFD，选择固定中模型 |
| CDS-M | 固定Model-M | 协同部署，固定中模型 |
| Random-M | 固定Model-M | 随机部署，固定中模型 |
| Greedy-M | 固定Model-M | 贪心部署，固定中模型 |

## 5. 指标计算

### 5.1 延迟计算

```python
# 端到端延迟 = Σ(排队延迟 + 处理延迟) + Σ通信延迟

# M/M/C 模型 (使用选定版本的 μ)
rho = lambda_ / (num_instances * mu_version)
Wq = erlang_c(rho) * rho / (num_instances * mu_version * (1 - rho))
W = Wq + 1000 / mu_version  # 转换为ms
```

### 5.2 精度计算

```python
# 每条链的精度 = 链中各服务精度的乘积 (或其他聚合方式)
chain_accuracy = product(service_versions[s].accuracy for s in chain.services)

# 系统平均精度 = 加权平均 (按到达率)
avg_accuracy = sum(c.arrival_rate * chain_accuracy for c in chains) / sum(c.arrival_rate)
```

## 9. 关键公式

### 9.1 M/M/C 排队模型

```
服务强度:        ρ = λ / (c × μ)

等待概率:        Pw = Erlang-C(rho, c)

平均队列长度:    Lq = ((cρ)^c × ρ × Pw) / (c! × (1-ρ)²)

平均等待时间:    Wq = Lq / λ  (ms)

平均响应时间:    W = Wq + 1000/μ  (ms)
```

### 9.2 端到端延迟

```
对于服务链 [s1, s2, ..., sk]:

总延迟 = Σ(排队延迟_i + 处理延迟_i) + Σ(通信延迟_i)

其中:
- 排队延迟_i = M/M/C(λ_i, μ_version_i, c_i).Wq
- 处理延迟_i = 1000 / μ_version_i
- 通信延迟_i = 节点间最短路径延迟 (若跨节点)
```

### 9.3 链精度

```
chain_accuracy = ∏_{i=1}^{k} accuracy(service_i_version)

avg_accuracy = Σ(λ_chain × chain_accuracy) / Σ(λ_chain)
```
