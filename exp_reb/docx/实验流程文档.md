# 服务部署与路由实验流程文档

## 1. 实验整体流程

```
┌─────────────────────────────────────────────────────────────────┐
│                          输入配置                                │
│                                                                 │
│  拓扑配置: 节点数、CPU/GPU资源、链路延迟                         │
│  服务配置: 服务数、CPU/GPU需求、处理速率                         │
│  服务链配置: 链数、到达率、链长、延迟约束                        │
│  算法配置: 部署算法、路由算法                                    │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                      Step 1: 数据生成                           │
│                                                                 │
│  DataGenerator                                                  │
│  ├── generate_topology()  → Topology对象                       │
│  ├── generate_services()   → Dict[str, MicroService]          │
│  └── generate_chains()     → List[ServiceChain]                │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                      Step 2: 算法执行                           │
│                                                                 │
│  ExperimentRunner.run_single()                                  │
│                                                                 │
│  输入: 拓扑 + 服务配置 + 服务链 + 算法名称                        │
│  输出: ExperimentResult                                          │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │                    部署算法执行                             │ │
│  │  DeploymentAlgorithm.deploy()                              │ │
│  │  → 输出 DeploymentPlan {(s,n): count}                      │ │
│  └─────────────────────────────────────────────────────────────┘ │
│                              ↓                                  │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │                   排队网络分析                               │ │
│  │  QueueingNetworkAnalyzer.calc_chain_latency()              │ │
│  │  → M/M/C公式计算每条链的延迟                                │ │
│  └─────────────────────────────────────────────────────────────┘ │
│                              ↓                                  │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │                    指标汇总                                  │ │
│  │  avg_latency / success_rate / resource_utilization / cost  │ │
│  └─────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│                      Step 3: 结果输出                           │
│                                                                 │
│  ├── results.json    # 详细实验结果                            │
│  ├── comparison.csv  # 算法对比表格                            │
│  └── figures/        # 可视化图表                              │
└─────────────────────────────────────────────────────────────────┘
```

## 2. 详细步骤说明

### Step 1: 数据生成

```python
# main.py

from experiments.generator import DataGenerator

# 创建生成器
gen = DataGenerator(seed=42)

# 拓扑
topology = gen.generate_topology({
    "num_nodes": 20,
    "cpu_cores_range": [8, 64],
    "gpu_memory_range": [0, 16384],    # 包含GPU
    "gpu_equipped_nodes_ratio": 0.5,  # 50%节点有GPU
    "link_delay_range": [0.1, 5.0],
})

# 服务配置 (包含GPU需求)
services = gen.generate_services({
    "num_services": 10,
    "num_gpu_services": 3,             # 前3个服务需要GPU
    "cpu_per_instance_range": [1, 4],
    "gpu_per_instance_range": [0, 4096],  # GPU内存需求
    "processing_rate_range": [5, 20],
})

# 服务链
chains = gen.generate_chains({
    "num_chains": 5,
    "arrival_rate_range": [1, 50],
    "chain_length_range": [2, 5],
    "max_latency": 100.0,
}, services)
```

**生成结果示例**:

```
Topology:
  - 20个节点，10个有GPU (n0-n9)
  - 全连接拓扑，链路延迟 0.1-5.0ms

Services (10个):
  - s0-s2: 需要GPU (GPU: 1024-4096 MB)
  - s3-s9: 不需要GPU

Chains (5条):
  - c0: [s0, s3, s5], λ=10, max_latency=100ms
  - c1: [s1, s4], λ=25, max_latency=80ms
  - ...
```

### Step 2: 单次实验执行

```python
from experiments.runner import ExperimentRunner

runner = ExperimentRunner(config)

# 执行单次实验
result = runner.run_single(
    deployment_name="ffd",
    topology=topology,
    services=services,
    chains=chains
)
```

#### 2.1 部署算法执行

```
FFD算法流程:

输入: 拓扑(20节点) + 服务(10个) + 链(5条)

Step 1: 计算每个服务的资源需求
  s0: λ总和=35, 需要5实例, CPU=10, GPU=5120MB
  s1: λ总和=25, 需要3实例, CPU=6, GPU=3072MB
  s2: λ总和=20, 需要3实例, CPU=6, GPU=2048MB
  s3-s9: 无GPU需求...

Step 2: 按总资源量降序
  s0 (GPU需求最大) > s1 > s2 > s3 > ...

Step 3: 贪心选择第一个满足条件的节点
  s0 → n0 (有GPU, 剩余4096MB) ✓
  s1 → n1 (有GPU, 剩余3072MB) ✓
  s2 → n2 (有GPU, 剩余2048MB) ✓
  s3 → n0 (剩余CPU足够) ✓
  ...

输出: DeploymentPlan
  {
    (s0, n0): 5,
    (s1, n1): 3,
    (s2, n2): 3,
    (s3, n0): 2,
    (s4, n3): 1,
    ...
  }
```

#### 2.2 排队网络分析

```python
analyzer = QueueingNetworkAnalyzer(topology, services)

# 对每条链计算延迟
for chain in chains:
    latency = analyzer.calc_chain_latency(chain, plan)
```

## 3. 关键公式

### 3.1 M/M/C 排队模型

```
服务强度:        ρ = λ / (c × μ)

等待概率:        Pw = ((cρ)^c / c!) / (Σ_{k=0}^{c-1} (cρ)^k/k! + (cρ)^c/(c!×(1-ρ)))

平均队列长度:    Lq = ((cρ)^c × ρ × Pw) / (c! × (1-ρ)²)

平均等待时间:    Wq = Lq / λ  (ms)

平均响应时间:    W = Wq + 1/μ  (ms)
```

### 3.2 端到端延迟

```
对于服务链 [s1, s2, ..., sk]:

总延迟 = Σ(排队延迟_i + 处理延迟_i) + Σ(通信延迟_i)

其中:
- 排队延迟_i = M/M/C模型的Wq
- 处理延迟_i = 1/μ_i
- 通信延迟_i = 两节点间最短路径延迟 (若跨节点)
```

### 3.3 资源利用率

```
节点CPU利用率 = 已用CPU / 总CPU

节点GPU利用率 = 已用GPU / 总GPU

系统平均利用率 = Σ各节点利用率 / 节点数
```

## 5. 典型实验场景

### 场景1: 静态部署对比

```
目的: 对比各算法在固定负载下的表现

参数:
- 20节点数据中心
- 10个微服务 (3个GPU服务)
- 5条服务链
- 到达率固定为 [1, 50]

对比算法:
- FFD, CDS, Random, Greedy, GRLS, Our

输出:
- 平均延迟排名
- 成功率排名
- 资源利用率对比
```

### 场景2: 到达率扰动

```
目的: 测试各算法对负载变化的鲁棒性

参数变化:
- 到达率: [1,20] → [1,400] (5个档位)

固定:
- 节点数=20, 服务数=10, 链数=5

输出:
- 各到达率下各算法的延迟曲线
- 找出各算法的稳定工作区间
```

### 场景3: GPU服务比例变化

```
目的: 测试GPU资源紧张时各算法的表现

参数变化:
- GPU服务比例: 10%, 30%, 50%, 70%, 90%

固定:
- 总服务数=10
- GPU节点比例=50%

输出:
- GPU利用率随比例变化曲线
- 各算法在GPU紧张时的表现
```

### 场景4: 链长变化

```
目的: 测试服务链长度对算法的影响

参数变化:
- 链长: 2, 3, 4, 5, 6, 7, 8

输出:
- 延迟随链长增长的趋势
- 通信延迟占比变化
```
