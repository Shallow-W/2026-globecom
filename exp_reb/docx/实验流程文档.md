## 2. 模型搜索 (核心步骤)

### 2.1 Excel模型库结构

```python
# Excel表结构

# - model_params: P_model (参数量, 内存占用)

```

### 2.2 动态算力红线

根据论文公式(3):

```python
def calc_F_max(C_max, lambda_t, T_SLA):
    """
    动态算力红线 (F_max)

    F_max(t) = C_max / (λ(t) + 1/T_SLA)

    流量λ越大 → 算力红线越收缩 → 只能选轻量模型
    """
    return C_max / (lambda_t + 1/T_SLA)
```

### 2.3 效用函数

根据论文公式(10):

```python
def calc_utility(model, w1, w2, w3):
    """
    效用函数

    U(a_k) = w1(t)·S_proxy - w2(t)·F_flops - w3(t)·P_model

    动态权重 (公式11):
    - w2(t) = α·exp(max(0, λ-λ_th)/λ_th)  # 流量惩罚
    - w3(t) = β·(M_used/M_total)             # 内存惩罚
    - w1(t) = 1/(1+w2+w3)                    # 归一化精度权重
    """
    return (w1 * model['S_proxy']
            - w2 * model['F_flops']
            - w3 * model['P_model'])
```

### 2.4 模型搜索流程

```python
class ModelSearcher:
    """模型搜索器 - Our算法的核心组件"""

    def __init__(self, excel_path):
        self.model_tables = {}  # {task_type: DataFrame}

    def load_models(self, task_type):
        """从Excel加载指定任务类型的模型库"""
        self.model_tables[task_type] = pd.read_excel(self.excel_path, sheet_name=task_type)

    def search(self, task_type, M_max, C_max, lambda_t, T_SLA):
        """
        根据当前节点状态搜索候选模型

        Args:
            task_type: 任务类型 (class_scene等)
            M_max: 节点可用内存
            C_max: 节点峰值算力
            lambda_t: 当前到达率
            T_SLA: SLA延迟约束

        Returns:
            List[Model]: 满足约束的候选模型列表
        """
        # Step 1: 计算动态算力红线
        F_max = C_max / (lambda_t + 1/T_SLA)

        # Step 2: 从Excel获取候选模型
        candidates = self.model_tables[task_type]

        # Step 3: 硬约束过滤
        filtered = candidates[
            (candidates['model_params'] <= M_max) &   # 内存约束
            (candidates['flops'] <= F_max)              # 算力约束
        ]

        # Step 4: 计算动态权重
        w2 = self.calc_w2(lambda_t)
        w3 = self.calc_w3(M_used, M_total)
        w1 = 1 / (1 + w2 + w3)

        # Step 5: 计算效用函数并排序
        filtered['utility'] = filtered.apply(
            lambda m: calc_utility(m, w1, w2, w3), axis=1
        )
        filtered = filtered.sort_values('utility', ascending=False)

        return filtered.to_dict('records')
```

## 3. Our算法详解

### 3.1 Deployment (部署)

```python
class OurDeployment:
    """
    Our部署算法 - 根据globecom.pdf的启发式调度

    三阶段流水线:
    1. 硬约束过滤
    2. 动态启发式加权选择
    3. 输出部署方案
    """

    def deploy(self, topology, services, chains, model_searcher):
        deployment_plan = DeploymentPlan()

        for node in topology.nodes:
            # 获取节点状态
            M_max = node.available_memory
            C_max = node.compute_capability
            lambda_t = self.get_current_load(node)  # 当前到达率

            for chain in chains:
                task_type = chain.task_type

                # 搜索候选模型
                candidates = model_searcher.search(
                    task_type=task_type,
                    M_max=M_max,
                    C_max=C_max,
                    lambda_t=lambda_t,
                    T_SLA=chain.max_latency
                )

                if candidates:
                    # 选择效用最大的模型
                    best_model = candidates[0]
                    # 部署到节点
                    deployment_plan.add(
                        service_id=chain.service_id,
                        node_id=node.node_id,
                        model=best_model,
                        version_id=best_model['architecture']
                    )

        return deployment_plan
```

### 3.2 Routing (路由)

```python
class OurRouting:
    """
    Our路由算法

    策略: 基于负载感知的路由
    - 优先选择当前负载较低的节点
    - 考虑路径延迟
    - 避免过载
    """

    def route(self, chain, deployment_plan, topology):
        """
        为服务链确定路由路径

        Returns:
            RoutingPlan: {service_id: [(node_id, probability), ...]}
        """
        routing = {}

        for service_id in chain.services:
            # 获取部署了该服务的所有节点
            candidate_nodes = deployment_plan.get_nodes_for_service(service_id)

            if not candidate_nodes:
                continue

            # 计算各节点的路由权重
            weights = []
            for node_id in candidate_nodes:
                # 负载越低、路径越短，权重越高
                load = self.get_node_load(node_id)
                path_delay = self.get_path_delay(chain.entry_node, node_id)
                weight = 1.0 / (load * path_delay + 1e-6)
                weights.append((node_id, weight))

            # 归一化权重
            total = sum(w for _, w in weights)
            routing[service_id] = [(n, w/total) for n, w in weights]

        return RoutingPlan(routing)
```

### 3.3 完整Our算法流程

```python
class OurAlgorithm:
    """
    Our完整算法 (部署 + 路由)

    对应globecom.pdf的云边协同按需调度框架:
    - 模型搜索 ≈ 零成本代理评估
    - Deployment ≈ 启发式加权选择部署
    - Routing ≈ 请求路由
    """

    def __init__(self, config):
        self.model_searcher = ModelSearcher(config['excel_path'])
        self.deployment = OurDeployment()
        self.routing = OurRouting()

    def solve(self, topology, services, chains):
        # Step 1: 加载模型库
        for task_type in set(c.task_type for c in chains):
            self.model_searcher.load_models(task_type)

        # Step 2: 部署
        deployment_plan = self.deployment.deploy(
            topology, services, chains, self.model_searcher
        )

        # Step 3: 路由
        routing_plan = {}
        for chain in chains:
            routing_plan[chain.chain_id] = self.routing.route(
                chain, deployment_plan, topology
            )

        return ExperimentResult(
            deployment_plan=deployment_plan,
            routing_plan=routing_plan
        )
```

## 5. 排队网络分析

### 5.1 FLOPs → μ 转换

```python
def flops_to_mu(F_flops, C_max):
    """
    将FLOPs转换为服务率μ

    μ = C_max / F_flops

    FLOPs越大 → μ越小 → 处理越慢
    """
    return C_max / F_flops if F_flops > 0 else float('inf')
```

### 5.2 延迟计算

```python
class QueueingAnalyzer:
    def calc_chain_latency(self, chain, deployment_plan, topology):
        total_delay = 0.0

        for i, service_id in enumerate(chain.services):
            # 获取部署的模型
            model = deployment_plan.get_model(service_id)

            # 计算μ
            mu = flops_to_mu(model['flops'], model['C_max'])

            # M/M/C排队
            instances = deployment_plan.get_instances(service_id)
            mmc = MMCQueue(
                arrival_rate=chain.arrival_rate,
                service_rate=mu,
                num_servers=instances
            )

            # 排队延迟 + 处理延迟
            total_delay += mmc.avg_response_time()

            # 通信延迟
            if i < len(chain.services) - 1:
                next_node = deployment_plan.get_node(service_id)
                curr_node = deployment_plan.get_node(chain.services[i+1])
                total_delay += topology.get_communication_delay(curr_node, next_node)

        return total_delay
```

### 场景2: 流量扰动

```
λ变化: [1,20] → [1,100] → [1,400]

观察:
- λ低时: Our选择高精度模型 (高S_proxy)
- λ高时: Our自动降级选择轻量模型 (低F_flops)
- Baseline固定中等模型，高负载时排队严重
```

### 场景3: 内存约束变化

```
M_max变化: 16GB → 8GB → 4GB

观察:
- 内存紧张时: Our选择小模型 (低P_model)
- Baseline可能因内存不足而失败
```
